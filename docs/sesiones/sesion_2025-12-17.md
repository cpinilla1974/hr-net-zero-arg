### 1. Expansión de Datos Históricos de Argentina

#### Problema Identificado
- Base de datos inicial contenía solo **3 años de datos** para Argentina (2015, 2020, 2023)
- Insuficiente para análisis de tendencias y trayectorias

#### Solución Implementada

**Búsqueda de datos en correos de FICEM**:
- Se realizó búsqueda en correos de `ficem.org`
- Se identificó email "Indicadores consolidados AFCP"
- Se descargó archivo: `Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx`

**Extracción de datos**:
- **Archivo fuente**: `/home/cpinilla/storage/ficem_bd/Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx`
- **Período extraído**: 2010-2024 (15 años)
- **Indicadores extraídos**:
  1. Factor clínker (fila 20) - Convertido a ratio (÷100)
  2. Energía térmica (fila 22) - Convertido a MJ/t (×1000)
  3. Tasa combustibles alternativos (fila 32) - En porcentaje
  4. Tasa biomasa (fila 33) - En porcentaje
  5. Emisiones netas CO₂ (fila 53) - En kg CO₂/t cemento

**Código de extracción**:
```python
import pandas as pd
import sqlite3

# Leer archivo Excel AFCP
df = pd.read_excel('Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx', sheet_name='Tabla_Consolidada')

# Mapeo de filas a indicadores
row_to_indicator = {
    20: ('clinker_cement_ratio', '%', 0.01),      # Dividir por 100
    22: ('thermal_energy_mj_clinker', 'MJ/t clinker', 1000),  # Multiplicar por 1000
    32: ('alternative_fuel_rate', '%', 1),
    33: ('biomass_rate', '%', 1),
    53: ('net_co2_kg_cement', 'kg CO2/t cement', 1),
}

# Insertar en SQLite
conn = sqlite3.connect('benchmarking.db')
cursor = conn.cursor()

for row_num, (indicator_code, unit, multiplier) in row_to_indicator.items():
    for col_idx in range(6, 21):  # Columnas 2010-2024
        year = 2010 + (col_idx - 6)
        value = df.iloc[row_num, col_idx] * multiplier

        cursor.execute("""
            INSERT INTO gnr_indicators (region, indicator, year, value, unit)
            VALUES (?, ?, ?, ?, ?)
        """, ('Argentina', indicator_code, year, value, unit))

conn.commit()
```

**Resultado**:
- **68 nuevos registros** insertados en la base de datos
- Datos de Argentina expandidos de **3 a 15 años**
- Cobertura completa 2010-2024 para 5 indicadores clave

## Decisiones Técnicas

### ¿Por qué cambiar a export default?

**Antes**:
```typescript
export function ProtectedRoute() {}
// Importar: import { ProtectedRoute } from "..."
```

**Después**:
```typescript
export default function ProtectedRoute() {}
// Importar: import ProtectedRoute from "..."
```

**Razón**: Consistencia con el patrón de Next.js donde las páginas usan default export.

### ¿Por qué allowedRoles en array?

**Razón**: Flexibilidad para múltiples roles. Ejemplo:
```typescript
<ProtectedRoute allowedRoles={["INFORMANTE_EMPRESA", "ADMIN_PROCESO"]}>
```

### ¿Por qué FormData manual en uploadFile?

**Razón**: El cliente API genérico usa `Content-Type: application/json`. Para archivos necesitamos `multipart/form-data`, por lo que usamos fetch nativo.

**8.5. Integración con API**

**useEffect principal**: Carga datos de todos los indicadores

```typescript
useEffect(() => {
  const fetchIndicatorData = async () => {
    // Construir lista de todos los indicadores
    const indicatorsList = INDICATORS.map(ind => ind.id).join(',');

    const res = await fetch(
      `/api/benchmarking?action=comparison&year=${selectedYear}&indicators=${indicatorsList}`
    );

    const data = await res.json();

    // Guardar valores de Argentina para cada indicador
    const argValues: Record<string, number> = {};
    INDICATORS.forEach(indicator => {
      const indicatorData = data.comparison?.[indicator.id] || [];
      const argData = indicatorData.find(d => d.region === 'Argentina');
      if (argData) {
        argValues[indicator.id] = argData.value * indicator.multiplier;
      }
    });
    setArgentinaValues(argValues);

    // Cargar datos de comparación para el indicador seleccionado
    const selectedData = data.comparison?.[selectedIndicator] || [];
    const regions = selectedData.map(d => ({
      region: d.region,
      value: d.value * (indicator?.multiplier || 1)
    }));
    setIndicatorComparison(regions);
  };

  fetchIndicatorData();
}, [selectedYear, selectedIndicator]);
```

**Optimización**:
- Una sola llamada a la API carga los 6 indicadores
- Los datos se actualizan reactivamente cuando cambia el año o el indicador
- Reduce las llamadas a la API comparado con múltiples useEffect separados
