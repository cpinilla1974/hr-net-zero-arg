### 1. Expansión de Datos Históricos de Argentina

#### Problema Identificado
- Base de datos inicial contenía solo **3 años de datos** para Argentina (2015, 2020, 2023)
- Insuficiente para análisis de tendencias y trayectorias

#### Solución Implementada

**Búsqueda de datos en correos de FICEM**:
- Se realizó búsqueda en correos de `ficem.org`
- Se identificó email "Indicadores consolidados AFCP"
- Se descargó archivo: `Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx`

**Extracción de datos**:
- **Archivo fuente**: `/home/cpinilla/storage/ficem_bd/Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx`
- **Período extraído**: 2010-2024 (15 años)
- **Indicadores extraídos**:
  1. Factor clínker (fila 20) - Convertido a ratio (÷100)
  2. Energía térmica (fila 22) - Convertido a MJ/t (×1000)
  3. Tasa combustibles alternativos (fila 32) - En porcentaje
  4. Tasa biomasa (fila 33) - En porcentaje
  5. Emisiones netas CO₂ (fila 53) - En kg CO₂/t cemento

**Código de extracción**:
```python
import pandas as pd
import sqlite3

# Leer archivo Excel AFCP
df = pd.read_excel('Rep_Sostenibilidad_CONSOLIDADO_AFCP.xlsx', sheet_name='Tabla_Consolidada')

# Mapeo de filas a indicadores
row_to_indicator = {
    20: ('clinker_cement_ratio', '%', 0.01),      # Dividir por 100
    22: ('thermal_energy_mj_clinker', 'MJ/t clinker', 1000),  # Multiplicar por 1000
    32: ('alternative_fuel_rate', '%', 1),
    33: ('biomass_rate', '%', 1),
    53: ('net_co2_kg_cement', 'kg CO2/t cement', 1),
}

# Insertar en SQLite
conn = sqlite3.connect('benchmarking.db')
cursor = conn.cursor()

for row_num, (indicator_code, unit, multiplier) in row_to_indicator.items():
    for col_idx in range(6, 21):  # Columnas 2010-2024
        year = 2010 + (col_idx - 6)
        value = df.iloc[row_num, col_idx] * multiplier

        cursor.execute("""
            INSERT INTO gnr_indicators (region, indicator, year, value, unit)
            VALUES (?, ?, ?, ?, ?)
        """, ('Argentina', indicator_code, year, value, unit))

conn.commit()
```

**Resultado**:
- **68 nuevos registros** insertados en la base de datos
- Datos de Argentina expandidos de **3 a 15 años**
- Cobertura completa 2010-2024 para 5 indicadores clave

## Decisiones Técnicas

### ¿Por qué cambiar a export default?

**Antes**:
```typescript
export function ProtectedRoute() {}
// Importar: import { ProtectedRoute } from "..."
```

**Después**:
```typescript
export default function ProtectedRoute() {}
// Importar: import ProtectedRoute from "..."
```

**Razón**: Consistencia con el patrón de Next.js donde las páginas usan default export.

### ¿Por qué allowedRoles en array?

**Razón**: Flexibilidad para múltiples roles. Ejemplo:

### ¿Por qué FormData manual en uploadFile?

**Razón**: El cliente API genérico usa `Content-Type: application/json`. Para archivos necesitamos `multipart/form-data`, por lo que usamos fetch nativo.

```typescript
useEffect(() => {
  const fetchIndicatorData = async () => {
    // Construir lista de todos los indicadores
    const indicatorsList = INDICATORS.map(ind => ind.id).join(',');

    const res = await fetch(
      `/api/benchmarking?action=comparison&year=${selectedYear}&indicators=${indicatorsList}`
    );

    const data = await res.json();

    // Guardar valores de Argentina para cada indicador
    const argValues: Record<string, number> = {};
    INDICATORS.forEach(indicator => {
      const indicatorData = data.comparison?.[indicator.id] || [];
      const argData = indicatorData.find(d => d.region === 'Argentina');
      if (argData) {
        argValues[indicator.id] = argData.value * indicator.multiplier;
      }
    });
    setArgentinaValues(argValues);

    // Cargar datos de comparación para el indicador seleccionado
    const selectedData = data.comparison?.[selectedIndicator] || [];
    const regions = selectedData.map(d => ({
      region: d.region,
      value: d.value * (indicator?.multiplier || 1)
    }));
    setIndicatorComparison(regions);
  };

  fetchIndicatorData();
}, [selectedYear, selectedIndicator]);
```

- Reduce las llamadas a la API comparado con múltiples useEffect separados

### Decisiones Técnicas

#### ¿Por qué array en lugar de diccionario?

**Opción descartada**:
```json
{
  "1": { "url": "...", "filename": "..." },
  "2": { "url": "...", "filename": "..." }
}
```

**Opción elegida**:
```json
[
  { "planta_id": 1, "url": "...", "filename": "..." },
  { "planta_id": 2, "url": "...", "filename": "..." }
]
```

**Razón**:
- Más fácil de iterar en frontend
- Mejor soporte para búsquedas (`.find()`)
- Consistente con otros arrays en el modelo (validaciones, comentarios)
- Permite agregar metadatos sin cambiar estructura

#### ¿Por qué modo fallback?

**Razón**: El usuario tenía reunión en 20 minutos y necesitaba demo funcional sin backend. El fallback permite:
- Desarrollo local sin dependencias
- Testing de UI sin API
- Demos rápidas
- Graceful degradation

**Comportamiento**:
1. Intenta usar API si está disponible
2. Si falla, usa datos mock
3. Limpia mensaje de error para no alarmar al usuario
4. Carga plantas demo según `empresa_id`

### 13.2. Selector de Año Dinámico por Indicador

#### Problema Identificado

El selector de año mostraba todos los años de 1990-2023, pero no todos los indicadores tienen datos para todos los años. Por ejemplo:
- `power_consumption_kwh_cement` solo tiene datos desde 2012
- `alternative_fuel_rate` solo tiene datos desde 2013

Esto causaba confusión al seleccionar años sin datos para el indicador actual.

#### Solución Implementada

**Archivo**: `app/src/app/benchmarking/page.tsx` (líneas 229-250)

**useEffect con dependencia en selectedIndicator**:
```typescript
// Cargar años disponibles desde Argentina para el indicador seleccionado
useEffect(() => {
  const fetchAvailableYears = async () => {
    try {
      const res = await fetch(`/api/benchmarking?action=time_series&indicator=${selectedIndicator}&regions=Argentina`);
      if (res.ok) {
        const data = await res.json();
        const argentinaData = data.series?.Argentina || [];
        const years = argentinaData.map((d: { year: number }) => d.year).sort((a: number, b: number) => b - a);
        setAvailableYears(years);

        // Si el año seleccionado no está disponible, seleccionar el más reciente
        if (years.length > 0 && !years.includes(selectedYear)) {
          setSelectedYear(years[0]);
        }
      }
    } catch (error) {
      console.error("Error fetching available years:", error);
    }
  };

  fetchAvailableYears();
}, [selectedIndicator]); // ← Dependencia clave
```

**Comportamiento**:
1. Al hacer click en una tarjeta de indicador diferente
2. Se dispara el useEffect por cambio en `selectedIndicator`
3. Se consulta a la API solo los años con datos de Argentina para ese indicador
4. Se actualiza el array `availableYears`
5. El slider se ajusta automáticamente al nuevo rango
6. Si el año actual no existe para el nuevo indicador, salta al año más reciente disponible

### 13.6. Decisiones de Diseño

#### ¿Por qué slider en lugar de dropdown?

**Ventajas del slider**:
- Navegación más rápida entre años consecutivos
- Visualización del rango completo
- Mejor experiencia táctil en móviles
- Menos clicks requeridos

**Desventaja aceptada**:
- Más difícil saltar a años específicos lejanos (ej: de 1990 a 2023)
- Compensado con el badge que muestra el año actual

#### ¿Por qué no remover los indicadores sin comparación?

**Razones para mantenerlos**:
1. **Datos valiosos**: Argentina ha reportado estos indicadores por años
2. **Transparencia**: Mostrar lo que hay disponible
3. **Contexto educativo**: Usuario entiende qué indicadores son más comunes internacionalmente
4. **Futuro**: Puede que otras regiones reporten estos datos más adelante

**Alternativa descartada**: Ocultar las tarjetas
- Generaría confusión ("¿por qué solo 4 indicadores?")
- Perdería consistencia visual del grid de 6 columnas

#### ¿Por qué color ámbar en lugar de rojo?

**Razón**: No es un error, es una limitación de datos
- Rojo sugiere "algo está roto"
- Ámbar/amarillo sugiere "advertencia informativa"
- Mantiene tono neutral y educativo
